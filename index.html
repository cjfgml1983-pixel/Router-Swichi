PC IP 주소 설정 (가상 사설 IP 사용)** - PC 클릭 후 **[Desktop] -> [IP Configuration]** 메뉴로 이동합니다. - **왼쪽 PC 예시:** - IP Address: 192.168.10.100 - Subnet Mask: 255.255.255.0 - Default Gateway: 192.168.10.1 (연결된 라우터의 IP) - 오른쪽 PC/서버도 해당 네트워크 대역(192.168.20.0/24)에 맞게 IP와 게이트웨이(192.168.20.1)를 설정합니다. ### **Step 3. 장비(라우터/스위치) 기본 설정** 장비를 클릭하고 **[CLI]** 탭으로 이동하여 설정을 진행합니다. (라우터 초기 대화창은 no 입력) - **모드 진입:** enable (특권 실행 모드) -> configure terminal (글로벌 설정 모드) - **장비 이름 설정 (예: 3층 스위치 '폴리텍ax'):** hostname PolytechAX_3F *(※ 패킷 트레이서 한글 깨짐 및 오류 방지를 위해 영문/숫자/언더바 조합 사용을 권장합니다.)* - **패스워드 설정:** - 특권 모드 암호: enable password cisco - 원격 접속(Telnet) 암호: line vty 0 4 -> password cisco -> login -> exit - 콘솔 접속 암호: line console 0 -> password cisco -> login -> exit - **인터페이스 IP 설정 및 활성화 (라우터 왼쪽 네트워크 예시):** - interface fa 0/0 - ip address 192.168.10.1 255.255.255.0 - no shutdown (포트 활성화, 빨간불이 초록불로 변경됨) - *오른쪽 포트(fa 0/1)도 동일한 방식으로 192.168.20.1을 할당합니다.* ### **Step 4. 연결성 검증** - 통신 테스트를 위해 출발지 PC를 클릭하고 [Desktop] -> [Command Prompt]로 이동합니다. - 명령어 ping[도착지 IP]를 입력합니다. (예: ping 192.168.20.200) - Reply 응답이 정상적으로 오는지 확인하여 종단 간 네트워크 연결이 성공했는지 검증합니다. ### **Step 5. Simulation 모드로 패킷 흐름 관찰** - 패킷 트레이서 우측 하단의 **[Simulation]** 탭을 클릭합니다. - [Edit Filters]를 클릭하여 관찰할 프로토콜인 **ARP**와 **ICMP**만 체크합니다. - PC에서 ping 명령어를 내린 후, 시뮬레이션 패널의 **Play(재생)** 버튼을 클릭합니다. - 스위치의 Flooding 동작을 통한 ARP MAC 주소 확인 과정과, 라우터를 거쳐 다른 네트워크로 ICMP(ping) 패킷이 전달 및 회신되는 전체 과정을 시각적으로 관찰합니다. --- 1. 패킷트레이서 CLI 프롬프트 ```jsx enable configure terminal hostname PolytechAX_3F enable password cisco line console 0 password cisco login exit line vty 0 4 password cisco login exit interface vlan 1 ip address 192.168.30.2 255.255.255.0 no shutdown exit exit show running-config ``` 1. 위 코드를 드래그해서 복사(Ctrl + C)합니다. 2. 패킷 트레이서에서 해당 스위치를 클릭하고 **[CLI]** 탭으로 들어갑니다. 3.
